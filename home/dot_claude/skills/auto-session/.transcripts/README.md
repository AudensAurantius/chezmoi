# `.transcripts/`

Snapshot record of design conversations that shaped this skill. Two files per session:

- `<date>-<topic>.jsonl` — **authoritative** raw Claude Code session log. Byte-for-byte what CC persisted; no rendering, no paraphrase. Diff against this if a Markdown rendering looks off.
- `<date>-<topic>.md` — lossy human-readable rendering generated by `~/.claude/scripts/render-jsonl.py`. `thinking` blocks omitted; tool payloads truncated at 1500 chars; system/snapshot/attachment records skipped.

## Why committed

These transcripts are snapshot-in-time artifacts of design intent — useful for
(a) revisiting rejected alternatives without re-deriving them, (b) auditing
which decisions were made under which context, (c) catching cases where an
implementation drifted from its design rationale. Keeping them alongside the
skill, rather than in `tasks/`, preserves the link when the skill is
re-homed or exported.

## Why `chezmoiignore`d

These files are working-author material, not skill deliverables. `chezmoi
apply` skips them so the installed `~/.claude/skills/auto-session/` stays lean.
The feature-branch worktree (and anyone checking out the branch in the
canonical worktree) still sees them.

## Regenerating

```sh
python3 ~/.claude/scripts/render-jsonl.py <session.jsonl> <out.md>           # full render
python3 ~/.claude/scripts/render-jsonl.py --append <session.jsonl> <out.md>  # incremental
```

Find the JSONL for a live session under `~/.claude/projects/<project-slug>/<session-id>.jsonl`.
